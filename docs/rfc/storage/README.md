# HNChain Storage RFCs

Storage RFCs define persistence, state database, snapshots, pruning, archival
behavior, and storage engine abstraction.

Required future RFCs include:

- storage and state interfaces
- storage record format
- column or namespace model
- state database API
- snapshot format
- pruning rules
- archival node requirements
- storage proof integration
- backend compatibility requirements

Current RFC drafts:

- [Storage And State Interfaces](storage-state-interfaces.md)
