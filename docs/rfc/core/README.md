# HNChain Core RFC

Status: Draft

Version: 0.1.0

Core RFC documents define implementation-facing rules for shared protocol
primitives used across HNChain modules.

These RFCs must remain small and stable. Core primitives are depended on by
cryptography, serialization, state, consensus, storage, networking, RPC, CLI,
and SDK components.

Core RFCs must not define application-level behavior.
