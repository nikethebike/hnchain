# HNChain Interoperability RFCs

Interoperability RFCs define cross-chain interfaces, bridge adapter behavior,
message formats, external proof verification, and wrapped asset integration.

Required future RFCs include:

- Bridge API
- adapter lifecycle
- cross-chain message format
- wrapped asset mapping
- external event format
- proof verification profiles
- bridge registry
- failure isolation
- security review requirements

Interoperability components must not silently become part of HNChain core
consensus unless accepted by an explicit protocol specification.
