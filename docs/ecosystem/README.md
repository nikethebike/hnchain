# HNChain Ecosystem

Status: Draft

Version: 0.1.0

Date: 2026-07-18

## Purpose

HNChain ecosystem documentation describes official and third-party tools built
around the protocol.

The protocol should remain minimal. The ecosystem can be rich.

## Ecosystem Components

Initial ecosystem components:

- HN Wallet
- HN Explorer
- HN SDK
- HN CLI
- HN IDE integrations
- HN Package Registry
- HN Developer Portal
- HN Faucet
- HN Testnet
- HN Devnet
- HN Validator Toolkit
- HN Scan API
- HN Identity
- HN DNS
- HN Ecosystem Registry

## Boundary

Ecosystem tools must use open specifications and public APIs.

Official tools may provide reference behavior, but they must not become hidden
sources of protocol truth.

Independent implementations must remain possible.

## Certification

HNChain may define a voluntary compatibility certification program.

Conceptual labels:

```text
Compatible -> Tested -> Certified
```

Certification criteria must be public, reproducible, and based on documented
specifications and compatibility tests.
